🌳 树模型算法超参数详解

决策树 (Decision Tree)

核心超参数：

  • max_depth：树的最大深度，防止过拟合
  • min_samples_split：分裂所需最小样本数
  • min_samples_leaf：叶节点最小样本数
  • criterion：分裂标准（gini/entropy）
  • max_features：每次分裂考虑的特征数

调优建议：从简单开始，逐步增加复杂度

随机森林 (Random Forest)

核心超参数：

  • n_estimators：树的数量，通常100-500
  • max_depth：单棵树的最大深度
  • min_samples_split：分裂最小样本数
  • max_features：随机选择特征数
  • bootstrap：是否使用自助采样

调优建议：先调n_estimators，再调其他参数

GBDT (Gradient Boosting)

核心超参数：

  • n_estimators：提升树的数量
  • learning_rate：学习率，通常0.01-0.3
  • max_depth：单棵树的最大深度
  • subsample：子采样比例
  • min_samples_split：分裂最小样本数

调优建议：learning_rate和n_estimators需要平衡

AdaBoost

核心超参数：

  • n_estimators：弱学习器数量
  • learning_rate：学习率，控制权重更新
  • algorithm：算法类型（SAMME/SAMME.R）
  • base_estimator：基础学习器类型

调优建议：learning_rate通常设为1.0

XGBoost

核心超参数：

  • n_estimators：树的数量
  • learning_rate：学习率，通常0.01-0.3
  • max_depth：树的最大深度
  • subsample：行采样比例
  • colsample_bytree：列采样比例
  • reg_alpha：L1正则化参数
  • reg_lambda：L2正则化参数

调优建议：先调learning_rate和n_estimators

LightGBM

核心超参数：

  • n_estimators：树的数量
  • learning_rate：学习率
  • max_depth：树的最大深度
  • num_leaves：叶节点数量
  • feature_fraction：特征采样比例
  • bagging_fraction：样本采样比例
  • min_data_in_leaf：叶节点最小样本数

调优建议：num_leaves通常设为2^max_depth

🎯 超参数调优策略

  • 网格搜索：穷举所有参数组合，适合参数空间小的情况
  • 随机搜索：随机采样参数组合，适合参数空间大的情况
  • 贝叶斯优化：智能搜索最优参数，适合昂贵的目标函数
  • 分层调优：先调重要参数，再调次要参数

🤖 算法对比分析

算法 优势 劣势 适用场景
逻辑回归 计算快速，易于理解 非线性表现不佳，受特征相关性影响 初步建模，基线模型
SVM 处理非线性，泛化能力强 大规模数据计算复杂，高维特征要求高 中等规模数据集
决策树 非线性处理，可解释性好 容易过拟合，不稳定 特征重要性分析
随机森林 克服过拟合，处理大规模数据 模型复杂度高，计算资源需求大 生产环境，高精度要求

🔍 特征工程流程

1. 数据预处理

数据清洗：异常值检测和处理、重复数据删除、数据格式统一

数据平衡：SMOTE过采样、随机欠采样、权重调整

数据归一化：标准化、MinMax归一化、鲁棒归一化

2. 特征选择

方差阈值：移除低方差特征

相关性分析：移除高相关特征

特征重要性：基于树模型的特征选择

递归特征消除：逐步移除不重要特征

3. 特征转换

独热编码：处理分类变量

特征组合：创建交互特征

特征降维：PCA、LDA等

多项式特征：非线性特征生成

4. 特征工程最佳实践

时序特征：滑动窗口统计、时间差特征、周期性特征

业务特征：比率特征、分箱特征、聚合特征

特征验证：特征稳定性检验、重要性分析、相关性监控

🔍 LIME vs SHAP 技术对比

LIME (局部可解释性)

技术原理：在样本附近生成假数据，用简单模型拟合，突出局部特征的贡献

解释范围：局部解释（单个样本）

理论基础：局部线性近似

计算复杂度：中等（需要生成扰动样本）

稳定性：较低（每次运行结果可能不同）

适用场景：单个预测解释、模型调试

# LIME使用示例 from lime import lime_tabular explainer = lime_tabular.LimeTabularExplainer( training_data, feature_names=feature_names, class_names=class_names ) explanation = explainer.explain_instance( data_instance, model.predict_proba )

SHAP (全局可解释性)

技术原理：基于博弈论，计算每个特征对预测的平均贡献

解释范围：全局解释（整体模型）

理论基础：博弈论中的Shapley值

计算复杂度：较高（需要计算所有特征组合）

稳定性：较高（结果一致性好）

适用场景：特征重要性分析、模型理解

# SHAP使用示例 import shap explainer = shap.TreeExplainer(model) shap_values = explainer.shap_values(X_test) shap.summary_plot(shap_values, X_test)
